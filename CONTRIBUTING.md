# Contributing

Thanks for contributing to this project.

## Getting Started

- Read [README.md](/Users/marek/Work/Ognicki/pull-secrets-operator/README.md) and [AGENTS.md](/Users/marek/Work/Ognicki/pull-secrets-operator/AGENTS.md) before making changes.
- Keep changes focused on the task at hand.
- Prefer small pull requests that are easy to review.

## Development Expectations

- Follow the existing project structure and naming.
- Keep behavior changes documented when they affect API or operator semantics.
- Avoid speculative changes that are not clearly needed.
- Update examples or documentation when code changes would otherwise make them stale.

## Code and Test Expectations

- Keep Go code formatted and consistent with the existing codebase.
- Add or update tests when behavior changes.
- Run the relevant test suite before opening a pull request.

## Pull Requests

- Describe what changed and why.
- Mention any user-visible behavior changes.
- Call out follow-up work or known limitations when relevant.
- Be responsive to review feedback and keep discussion constructive.

## Communication

- Assume good intent.
- Prefer clear, respectful, and specific feedback.
- If a design or behavior is unclear, ask before making a broad change.
